Cloud Based. Everywhere and Everytime Availability.

A cloud-based web application with top-level a web responsive friendly interface and a RESTful API ensure the abstraction of details, storage of devices data and easy access to domotic devices. It also allows the access from other automation services which can process all the data collected by the devices on the mHouse Framework. More info

Home Server Mediated. Both Upstream and Downstream Communications Ensured.

A middleware/gateway application between the Cloud Platform and the domotic devices, where all the Framework's system is built upon. Devices are represented here as resources, allowing the two-way communication by using the RESTful API provided by the Cloud Platform and by providing its own RESTful APIs as well, namely an HTTP based one and a CoAP based one. More info

Endpoints for Low-Level Logic. Simple and Economic Devices upon the Physical Interface.

Endpoint configurable applications, running on the device with the attached physical sensors and actuators, abstract the whole physical low-level logic details. All the communications are ensured by the base application code that can be configured accordingly to each sensor, actuactor or physical scenario and all the interactions with the upper system layers are made through the RESTful CoAP API provided by the Home Server component. More info

About

The mHouse Framework, which name stands for "my House Framework" and is pronounced "mouse framework", is an open source Domotic Framework for the Internet of Things, developed by José Diogo Requeijo Dias during his master thesis on Eletrical and Computer Engeneering at Instituto Superior Técnico, Lisbon.
