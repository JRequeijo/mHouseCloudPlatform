Online Cloud HTTP RESTful API:

URI

services/

Method: GET

Action

Get all the Services for the authenticated user.

Possible Response Codes and Body

  • 200 OK - JSON object with all the Services.

Method: POST

Action

Add (create) a new Service for the authenticated user.

Request Body

JSON describing the new Service.

Required Fields
  • name - The name of the new Service. This field must be unique for the user.
Optional Fields
  • core_service_ref - The ID of a Core Service that the user wants to use as a “own”/custom service. If this field is provided and valid, the name of the new service will always be the name of the Core Service referenced by this field, disregarding the name field.

Possible Response Codes and Body

  • 200 OK - JSON object with all the Services of the authenticated user.
  • 400 BAD REQUEST - Something is wrong with the request.
